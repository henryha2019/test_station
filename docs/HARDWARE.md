# 硬件 · 接线 · 固定 (MG996R 伺服测试站)

被测件(DUT)= **MG996R 舵机**。判定 = **功能测试 AND 视觉检测**。
原则:**仪器只测量,主机做判定;所有节点共地 (common GND)**。

> 接线总览图:**[wiring_mg996r.svg](wiring_mg996r.svg)**

---

## 1. 需要哪些硬件 (BOM)

| 件 | 数量 | 作用 | 备注 |
|---|---|---|---|
| Arduino Uno | 1 | 功能测试仪:发 PWM 指令、读角度/电流 | 也可用 Nano |
| **AS5600 磁编码器模块** | 1 | 测舵机输出轴**实际角度**(I²C 0x36) | 配一颗**径向充磁**磁铁 |
| **INA219 电流传感器模块** | 1 | 测舵机**电流**(I²C 0x40) | 高边测量 |
| ESP32-CAM (AI-Thinker) | 1 | 拍照做视觉检测 | 烧录需 USB-TTL/FTDI |
| ESP32 开发板 + SSD1306 OLED | 1 | 工位 HMI 显示判定 | OLED I²C 128×64 |
| **5–6V 电源 ≥3A** | 1 | 给舵机供电 | MG996R 堵转可达 ~2.5A,**别用 Uno/USB 供电** |
| 3 针母头排线 (对插舵机插头) | 1 | 接触 DUT 的电源/地/信号 | 舵机自带标准 3 针公头 |
| 白光 LED + 限流电阻 | 数颗 | 扩散罩照明 | 由 5V 常亮 |
| 杜邦线/接线端子 | 若干 | | |
| 3D 打印件 | 一套 | 舵机巢 + 磁铁联轴器 + AS5600 座 + 相机支架 + 扩散罩 | 见第 3 节 |

> **关于探针(pogo):用减法。** 舵机自带标准 3 针插头,直接对插一个 3 针母座即可,
> 比弹簧探针更可靠——这个 DUT 不需要 pogo 针。

## 2. 怎么连接 (接线)

### 2.1 功能测试仪 (Arduino Uno)

舵机 3 根线:**信号(橙/黄)→ Uno D9**;**电源(红)→ 经 INA219 接 5–6V 电源正**;
**地(棕/黑)→ 公共地**。

| 电流路径 (高边测量) | 接法 |
|---|---|
| 5–6V 电源 (+) | → **INA219 `Vin+`** |
| INA219 `Vin-` | → **舵机电源线(红)** |
| 5–6V 电源 (−) | → 公共地(与 Uno GND、电源地相连) |

| INA219 / AS5600 (I²C，共用一条总线) | Uno |
|---|---|
| INA219 `VCC` | 5V |
| AS5600 `VCC` | **3V3**(AS5600 是 3.3V 器件) |
| 两者 `GND` | GND |
| 两者 `SDA` | A4 |
| 两者 `SCL` | A5 |

地址不冲突(AS5600=0x36，INA219=0x40)。**Uno ↔ 主机:USB**(串口 115200)。

> 注意:AS5600 用 3.3V 供电最稳。若你的 AS5600 模块板载稳压且 I²C 5V 容限,
> 也可 5V;不确定就用 3V3 + 同总线上 3.3V 上拉。

### 2.2 视觉 (ESP32-CAM)

| 引脚 | 接到 |
|---|---|
| `5V` / `GND` | **独立 5V**(相机电流大,别只靠 FTDI) |
| (Wi-Fi) | 与主机同一局域网,主机 `GET /capture` 取 JPEG (`vision.espcam_url`) |

烧录时(板上无 USB):`U0R→FTDI TX`、`U0T→FTDI RX`、`5V/GND` 对接、
**`GPIO0→GND` 进下载模式**,烧完断开并复位。

### 2.3 HMI (ESP32 + OLED)

| 连接 | 引脚 |
|---|---|
| ESP32 ↔ 主机 | USB 串口 115200 (`station.hmi_port`) |
| OLED `VCC`/`GND` | 3V3 / GND |
| OLED `SCL` | GPIO22 |
| OLED `SDA` | GPIO21 |

主机每测一台推一帧:`HMI,<serial>,<func>,<vision>,<class>,<final>,<fpy>`。

### 2.4 供电与共地(关键)

```
主机 USB ──┬── Arduino Uno
           └── ESP32 (HMI)

独立 5–6V ─┬── 舵机 (经 INA219)
(≥3A)      ├── ESP32-CAM
           └── 照明 LED

★ 所有 GND 必须连在一起(Uno、电源、INA219、AS5600、舵机)= common GND
```

## 3. 怎么固定 (夹具 / 安装)

视觉成功 80% 靠重复定位 + 受控照明;功能测试的精度靠**轴与编码器同心**。

1. **舵机巢 (3-2-1 定位 + 防呆)**:按 MG996R 外形(约 40.7×19.7×42.9mm)打印一个凹槽,
   贴住底面 + 两侧 + 一端;加一个**非对称特征**(防呆),装反就放不进。
   用一个**手动快夹(toggle clamp)** 把舵机压紧(v2.0 用手动最可靠;舵机自动夹紧
   属可选)。

2. **角度测量同轴 (核心机械点)**:
   - 在舵机输出轴(花键)上装一个打印的**联轴器**,顶端粘一颗**径向充磁磁铁**。
   - AS5600 板固定在舵机轴**正下方/正上方**的座里,磁铁中心对准芯片中心,
     **气隙 ~1–2mm**。
   - 这样舵机一转,AS5600 直接读到真实角度 → 测行程/转向/中位偏差。
   - (不想做联轴?替代:用相机光学读舵机臂角度,精度较低——见 vision/。)

3. **接触 DUT**:夹具上固定一个 **3 针母座**,座好舵机后把舵机插头对插进去
   (电源/地/信号)。无需探针。

4. **相机 + 扩散罩**:相机支架固定在**重复一致的工作距离**;扩散罩把舵机和相机罩住,
   几颗白光 LED 藏在扩散材料(白色 PETG/硫酸纸/乒乓球塑料)后 → 平整恒定光。
   每个循环取景与光照都相同。

> 上方两张渲染图为按真实板型绘制的**示意图**,非实物照片。实物引脚以各模块数据手册为准
> (Arduino UNO R3、AS5600、INA219、AI-Thinker ESP32-CAM、ESP32-WROOM-32、SSD1306)。

## 4. 软件里对应的地方

- 命令位置/限值窗口:`config/mg996r.json → functional`
- 串口号 / `espcam_url` / 传感器地址:`config/mg996r.json → station`
- 固件:`firmware/servo_tester_uno/`(功能仪)、`firmware/esp32_hmi/`(HMI)
- 上电后先 `python -m host.app --simulate --ui` 跑通仿真,再插硬件用 `--port`。
